Bảo mật ứng dụng Java: JWT Authentication và Authorization

Giải thích JWT, cách tạo/verify token, cấu hình Spring Security filter chain và phân quyền theo vai trò.

JWT là gì?

JWT (JSON Web Token) là chuỗi mã hóa gồm header, payload và signature. Dùng để xác thực stateless giữa client và server.

Flow xác thực

  1. Đăng nhập nhận access token
  2. Client gửi token trong Authorization header
  3. Filter verify token, set Authentication vào SecurityContext

Bộ lọc xác thực JWT (ví dụ rút gọn)

public class JwtAuthFilter extends OncePerRequestFilter {
  @Override
  protected void doFilterInternal(HttpServletRequest request, HttpServletResponse response, FilterChain chain)
      throws ServletException, IOException {
    String header = request.getHeader("Authorization");
    if (header != null && header.startsWith("Bearer ")) {
      String token = header.substring(7);
      // verify token -> set Authentication
      UsernamePasswordAuthenticationToken auth = new UsernamePasswordAuthenticationToken("user", null, List.of());
      SecurityContextHolder.getContext().setAuthentication(auth);
    }
    chain.doFilter(request, response);
  }
}

Phân quyền

Dùng roles/authorities, cấu hình antMatchers/http.authorizeHttpRequests cho endpoints tương ứng.

Refresh Token và TTL

  • Dùng access token ngắn hạn, refresh token dài hạn lưu an toàn (HTTP-only cookie)
  • Rotate refresh token và revoke khi nghi ngờ bị lộ

Tạo JWT (ý tưởng)

String token = Jwts.builder()
  .setSubject(username)
  .setIssuedAt(new Date())
  .setExpiration(Date.from(Instant.now().plusSeconds(900)))
  .signWith(SignatureAlgorithm.HS256, secret)
  .compact();
