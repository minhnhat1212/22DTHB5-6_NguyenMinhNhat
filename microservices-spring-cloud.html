Microservices Architecture với Spring Cloud

Giới thiệu service discovery (Eureka), API Gateway (Spring Cloud Gateway), config server và resilience (Resilience4j).

Các thành phần cốt lõi

  • Service Registry: Eureka giúp services tự đăng ký và tìm nhau
  • API Gateway: Điều phối traffic, auth, rate limit
  • Config Server: Quản lý cấu hình tập trung
  • Resilience: Circuit breaker, retry, bulkhead

application.yml cho Eureka Client

spring:
  application:
    name: order-service
eureka:
  client:
    service-url:
      defaultZone: http://localhost:8761/eureka/

Gợi ý kiến trúc

Tách domain theo bounded context, giao tiếp qua REST/Message broker, quan sát hệ thống với metrics/logs/tracing.

Observability & Resilience

  • Metrics (Micrometer + Prometheus), logs cấu trúc (JSON), tracing (OpenTelemetry)
  • Resilience4j: circuit breaker, retry, rate limiter

Resilience4j annotations

@Retry(name = "inventory")
@CircuitBreaker(name = "inventory", fallbackMethod = "fallback")
public Stock checkStock(String sku) { /* ... */ }

Hình minh họa / Sơ đồ

Kiến trúc microservices (giản lược)

[Client] → [API Gateway] → [Service A] ↔ [Service B]
                                   ↘ [Service C]
[Services] ↔ [Service Registry]   [Config Server]

Tài liệu tham khảo

  • Spring Cloud
  • Resilience4j
