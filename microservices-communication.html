Microservices Communication: Giao tiếp giữa các dịch vụ

Java 08/12/2024 5 phút đọc

Trong kiến trúc microservices, việc giao tiếp giữa các dịch vụ là một thách thức quan trọng. Có hai cách chính: synchronous và asynchronous communication.

1. Nó là gì?

Synchronous Communication: Các dịch vụ giao tiếp trực tiếp với nhau và chờ phản hồi ngay lập tức (HTTP/REST, gRPC).

Asynchronous Communication: Các dịch vụ giao tiếp thông qua message queues hoặc event streaming mà không cần chờ phản hồi ngay lập tức.

2. Nó giúp ích cho tôi ra sao khi sử dụng?

  • Scalability: Có thể scale từng dịch vụ độc lập
  • Resilience: Một dịch vụ lỗi không ảnh hưởng đến toàn bộ hệ thống
  • Technology Diversity: Mỗi dịch vụ có thể sử dụng công nghệ khác nhau
  • Team Independence: Các team có thể phát triển độc lập
  • Fault Isolation: Lỗi được cô lập trong từng dịch vụ

3. Cách sử dụng

3.1. Synchronous Communication

// Service Discovery với Eureka
@SpringBootApplication
@EnableEurekaClient
@RestController
public class UserServiceApplication {
    
    @Autowired
    private RestTemplate restTemplate;
    
    @Bean
    @LoadBalanced
    public RestTemplate restTemplate() {
        return new RestTemplate();
    }
    
    @GetMapping("/users/{id}/orders")
    public ResponseEntity> getUserOrders(@PathVariable Long id) {
        String orderServiceUrl = "http://order-service/orders/user/" + id;
        
        try {
            Order[] orders = restTemplate.getForObject(orderServiceUrl, Order[].class);
            return ResponseEntity.ok(Arrays.asList(orders));
        } catch (Exception e) {
            return ResponseEntity.status(HttpStatus.SERVICE_UNAVAILABLE)
                .body(Collections.emptyList());
        }
    }
}

3.2. Asynchronous Communication

// RabbitMQ Producer
@Service
public class OrderEventPublisher {
    
    @Autowired
    private RabbitTemplate rabbitTemplate;
    
    public void publishOrderCreated(Order order) {
        OrderEvent event = new OrderEvent();
        event.setOrderId(order.getId());
        event.setUserId(order.getUserId());
        event.setEventType("ORDER_CREATED");
        event.setTimestamp(Instant.now());
        
        rabbitTemplate.convertAndSend("order.exchange", "order.created", event);
    }
}

// RabbitMQ Consumer
@RabbitListener(queues = "order.created.queue")
public void handleOrderCreated(OrderEvent event) {
    // Xử lý event order được tạo
    System.out.println("Order created: " + event.getOrderId());
}

4. Kết luận

Khuyến nghị: Sử dụng synchronous communication cho các tác vụ cần phản hồi ngay lập tức, và asynchronous communication cho các tác vụ có thể xử lý sau.
