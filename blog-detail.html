Spring Boot: Xây dựng RESTful API từ cơ bản đến nâng cao

Hướng dẫn chi tiết cách xây dựng RESTful API với Spring Boot, từ cấu hình cơ bản đến các tính năng nâng cao như security, validation và testing.

Lý thuyết cơ bản

1. Spring Boot là gì?

Spring Boot là một framework Java được xây dựng trên nền tảng Spring Framework, giúp tạo ra các ứng dụng Spring một cách nhanh chóng và dễ dàng. Nó cung cấp auto-configuration, embedded servers và production-ready features.

2. RESTful API

REST (Representational State Transfer) là một kiến trúc phần mềm để thiết kế web services. RESTful API tuân theo các nguyên tắc:

  • Stateless: Mỗi request phải chứa đầy đủ thông tin cần thiết
  • Client-Server: Tách biệt giao diện người dùng và lưu trữ dữ liệu
  • Cacheable: Responses có thể được cache
  • Uniform Interface: Sử dụng HTTP methods chuẩn (GET, POST, PUT, DELETE)

3. Các Annotation quan trọng

@RestController

Kết hợp @Controller và @ResponseBody, tự động convert return value thành JSON/XML

@RequestMapping

Định nghĩa URL mapping cho controller methods

@Autowired

Dependency Injection - Spring tự động inject dependencies

@Service, @Repository, @Component

Các annotation để đánh dấu các class thuộc các layer khác nhau

Bước 3: Tạo Entity

User.java

@Entity
@Table(name = "users")
public class User {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;
    
    @NotBlank
    @Column(unique = true)
    private String username;
    
    @Email
    private String email;
    
    // Constructors, getters, setters
}

Bước 4: Tạo Repository

UserRepository.java

@Repository
public interface UserRepository extends JpaRepository<User, Long> {
    Optional<User> findByUsername(String username);
    Optional<User> findByEmail(String email);
    List<User> findByActiveTrue();
}

Bước 5: Tạo Service Layer

UserService.java

@Service
@Transactional
public class UserService {
    
    @Autowired
    private UserRepository userRepository;
    
    public List<User> getAllUsers() {
        return userRepository.findAll();
    }
    
    public User getUserById(Long id) {
        return userRepository.findById(id)
            .orElseThrow(() -> new UserNotFoundException("User not found"));
    }
    
    public User createUser(User user) {
        // Validation logic
        return userRepository.save(user);
    }
}

Bước 6: Tạo REST Controller

UserController.java

@RestController
@RequestMapping("/api/users")
@CrossOrigin(origins = "*")
public class UserController {
    
    @Autowired
    private UserService userService;
    
    @GetMapping
    public ResponseEntity<List<User>> getAllUsers() {
        List<User> users = userService.getAllUsers();
        return ResponseEntity.ok(users);
    }
    
    @GetMapping("/{id}")
    public ResponseEntity<User> getUserById(@PathVariable Long id) {
        User user = userService.getUserById(id);
        return ResponseEntity.ok(user);
    }
    
    @PostMapping
    public ResponseEntity<User> createUser(@Valid @RequestBody User user) {
        User createdUser = userService.createUser(user);
        return ResponseEntity.status(HttpStatus.CREATED).body(createdUser);
    }
}

Lý thuyết nâng cao

1. Exception Handling

Sử dụng @ControllerAdvice và @ExceptionHandler để xử lý exceptions một cách tập trung: 

@ControllerAdvice
public class GlobalExceptionHandler {
    
    @ExceptionHandler(UserNotFoundException.class)
    public ResponseEntity<ErrorResponse> handleUserNotFound(UserNotFoundException ex) {
        ErrorResponse error = new ErrorResponse("USER_NOT_FOUND", ex.getMessage());
        return ResponseEntity.status(HttpStatus.NOT_FOUND).body(error);
    }
}

2. Validation

Sử dụng Bean Validation để validate input data:

  • @NotNull, @NotBlank, @NotEmpty
  • @Email, @Pattern, @Size
  • @Min, @Max, @Range
  • Custom validators

3. Security

Spring Security cung cấp authentication và authorization:

  • JWT Authentication
  • Role-based Access Control
  • CORS Configuration
  • Password Encoding

4. Testing

Viết tests cho API endpoints:

  • Unit Tests với @MockBean
  • Integration Tests với @SpringBootTest
  • TestContainers cho database testing
  • MockMvc cho controller testing

Kết luận

Spring Boot giúp việc xây dựng RESTful API trở nên đơn giản và hiệu quả. Với auto-configuration và convention over configuration, bạn có thể tập trung vào business logic thay vì cấu hình phức tạp.

Những điểm quan trọng cần nhớ:

  • Luôn sử dụng layered architecture (Controller → Service → Repository)
  • Implement proper error handling và validation
  • Viết tests đầy đủ cho tất cả endpoints
  • Follow REST conventions và HTTP status codes
  • Document API với Swagger/OpenAPI
