Node.js: Xây dựng Server-side với JavaScript

Thiết lập web server với Express, routing cơ bản, kết nối MongoDB và cấu trúc project gọn gàng.

Express cơ bản

Tạo server, định nghĩa routes, middleware cho logging, JSON parsing và xử lý lỗi.

Express app tối giản

const express = require('express');
const app = express();
app.use(express.json());

app.get('/health', (req, res) => res.json({ status: 'ok' }));

app.use((err, req, res, next) => {
  console.error(err);
  res.status(500).json({ message: 'Internal error' });
});

app.listen(3000, () => console.log('Server running on 3000'));

Kết nối MongoDB

Dùng Mongoose để định nghĩa Schema/Model, CRUD đơn giản và validate dữ liệu.

Mongoose model và route CRUD

const mongoose = require('mongoose');
mongoose.connect(process.env.MONGO_URL);

const User = mongoose.model('User', new mongoose.Schema({
  name: { type: String, required: true }
}));

app.post('/users', async (req, res) => {
  const user = await User.create({ name: req.body.name });
  res.status(201).json(user);
});

Cấu trúc dự án

  • src/app.js: khởi tạo app và middleware
  • src/routes/*.js: định nghĩa endpoints
  • src/models/*.js: schema & model
  • src/controllers/*.js: tách controller logic

Bảo mật & vận hành

  • Ẩn credentials bằng biến môi trường, không commit .env
  • Dùng helmet, rate limiting, cors phù hợp origin
  • Log theo cấu trúc (pino/winston), healthcheck và readiness

Rate limit và Helmet

const helmet = require('helmet');
const rateLimit = require('express-rate-limit');
app.use(helmet());
app.use(rateLimit({ windowMs: 60_000, max: 100 }));

Hình minh họa / Sơ đồ

Kiến trúc Express + MongoDB

[Client] → [Express Routes] → [Controllers] → [Models] → [MongoDB]
