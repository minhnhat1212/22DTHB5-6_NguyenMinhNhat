Unit Testing và Integration Testing trong Java

Phân biệt unit/integration test, setup JUnit 5, Mockito và giới thiệu Testcontainers cho kiểm thử tích hợp.

Unit test

Tập trung vào một lớp/hàm riêng lẻ, dùng mocks để cô lập dependency, chạy nhanh.

JUnit 5 + Mockito

@ExtendWith(MockitoExtension.class)
class UserServiceTest {
  @Mock UserRepository repo;
  @InjectMocks UserService service;

  @Test
  void createUser_success() {
    when(repo.save(any())).thenAnswer(inv -> inv.getArgument(0));
    var res = service.create("alice");
    assertEquals("alice", res.getName());
  }
}

Integration test

Kiểm tra tương tác giữa các thành phần, có thể khởi động context hoặc dịch vụ phụ trợ (DB, MQ) bằng Testcontainers.

SpringBootTest với Testcontainers (ý tưởng)

@SpringBootTest
@Testcontainers
class UserApiIT {
  @Container static PostgreSQLContainer db = new PostgreSQLContainer<>("postgres:15");
  @Test void getUsers_ok() { /* call REST endpoint and assert */ }
}

Chiến lược test

  • Pyramid testing: nhiều unit test, ít integration, rất ít e2e
  • Test theo hành vi (Given-When-Then), tránh mock quá mức
  • CI: chạy test song song, báo cáo coverage
