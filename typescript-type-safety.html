TypeScript: JavaScript với Type Safety

Lợi ích của types, cách định nghĩa interface/type, generics và thiết lập project cơ bản.

TypeScript là một superset của JavaScript được phát triển bởi Microsoft, mang lại khả năng type checking tĩnh và các tính năng hiện đại cho JavaScript. Với TypeScript, bạn có thể viết code JavaScript an toàn hơn, dễ bảo trì hơn và có trải nghiệm phát triển tốt hơn.

Vì sao cần TypeScript?

Giảm lỗi runtime

TypeScript phát hiện lỗi ngay trong quá trình phát triển, trước khi code được chạy.

Auto-completion thông minh

IDE cung cấp gợi ý chính xác dựa trên type definitions.

Refactor an toàn

Thay đổi code với sự tự tin, TypeScript sẽ báo lỗi nếu có vấn đề.

Tài liệu hóa code

Types tự động tạo documentation cho API và cấu trúc dữ liệu.

Cài đặt và thiết lập

1. Cài đặt TypeScript

# Cài đặt global
npm install -g typescript

# Hoặc cài đặt local cho project
npm install --save-dev typescript

# Khởi tạo tsconfig.json
tsc --init

2. Cấu hình tsconfig.json cơ bản

{
  "compilerOptions": {
    "target": "ES2020",
    "module": "commonjs",
    "strict": true,
    "esModuleInterop": true,
    "skipLibCheck": true,
    "forceConsistentCasingInFileNames": true,
    "outDir": "./dist",
    "rootDir": "./src"
  },
  "include": ["src/**/*"],
  "exclude": ["node_modules", "dist"]
}

Các khối xây dựng cơ bản

1. Basic Types

// Primitive types
let name: string = "Nguyễn Minh Nhật";
let age: number = 25;
let isActive: boolean = true;
let data: any = "có thể là bất kỳ kiểu nào";

// Arrays
let numbers: number[] = [1, 2, 3, 4, 5];
let names: Array = ["Alice", "Bob", "Charlie"];

// Tuples
let person: [string, number] = ["Alice", 30];

// Enums
enum Status {
  PENDING = "pending",
  APPROVED = "approved",
  REJECTED = "rejected"
}

2. Interfaces và Type Aliases

// Interface
interface User {
  id: number;
  name: string;
  email: string;
  isActive?: boolean; // Optional property
  readonly createdAt: Date; // Readonly property
}

// Type alias
type ApiResponse = {
  data: T;
  success: boolean;
  message?: string;
  error?: string;
};

// Extending interfaces
interface AdminUser extends User {
  permissions: string[];
  canDelete: boolean;
}

3. Generics - Tính tái sử dụng cao

// Generic function
function identity(arg: T): T {
  return arg;
}

// Generic interface
interface Repository {
  findById(id: number): T | null;
  save(entity: T): T;
  delete(id: number): boolean;
}

// Generic class
class DataService {
  private data: T[] = [];
  
  add(item: T): void {
    this.data.push(item);
  }
  
  getAll(): T[] {
    return this.data;
  }
  
  find(predicate: (item: T) => boolean): T | undefined {
    return this.data.find(predicate);
  }
}

// Usage
const userService = new DataService();
const productService = new DataService();

Union Types và Type Guards

// Union types
type Status = "loading" | "success" | "error";
type StringOrNumber = string | number;

// Discriminated unions
type LoadingState = {
  status: "loading";
};

type SuccessState = {
  status: "success";
  data: any;
};

type ErrorState = {
  status: "error";
  error: string;
};

type AppState = LoadingState | SuccessState | ErrorState;

// Type guards
function isSuccessState(state: AppState): state is SuccessState {
  return state.status === "success";
}

function handleState(state: AppState) {
  switch (state.status) {
    case "loading":
      console.log("Đang tải...");
      break;
    case "success":
      console.log("Dữ liệu:", state.data);
      break;
    case "error":
      console.log("Lỗi:", state.error);
      break;
  }
}

Advanced Types

1. Utility Types

interface User {
  id: number;
  name: string;
  email: string;
  password: string;
  createdAt: Date;
}

// Partial - tất cả properties đều optional
type UserUpdate = Partial;

// Pick - chỉ lấy một số properties
type UserPublic = Pick;

// Omit - loại bỏ một số properties
type UserCreate = Omit;

// Record - tạo object type với key-value
type UserRoles = Record;

// Required - tất cả properties đều required
type UserRequired = Required;

2. Conditional Types

// Conditional type
type ApiResponse = T extends string 
  ? { message: T } 
  : { data: T };

// Mapped types
type Optional = {
  [P in keyof T]?: T[P];
};

// Template literal types
type EventName = `on${Capitalize}`;
type ClickEvent = EventName<'click'>; // "onClick"

Best Practices

Sử dụng strict mode

Bật strict mode trong tsconfig.json để có type checking nghiêm ngặt nhất.

Tránh sử dụng 'any'

Thay vì dùng 'any', hãy định nghĩa types cụ thể hoặc sử dụng 'unknown'.

Sử dụng interfaces cho objects

Interfaces phù hợp cho việc định nghĩa cấu trúc object, types cho unions và primitives.

Leverage type inference

Để TypeScript tự suy luận types khi có thể, chỉ khai báo explicit types khi cần thiết.

Ví dụ thực tế: API Service

// types/api.ts
export interface ApiResponse {
  data: T;
  success: boolean;
  message?: string;
  errors?: string[];
}

export interface User {
  id: number;
  name: string;
  email: string;
  avatar?: string;
  createdAt: string;
  updatedAt: string;
}

export interface CreateUserRequest {
  name: string;
  email: string;
  password: string;
}

export interface UpdateUserRequest {
  name?: string;
  email?: string;
  avatar?: string;
}

// services/userService.ts
export class UserService {
  private baseUrl: string;

  constructor(baseUrl: string = '/api') {
    this.baseUrl = baseUrl;
  }

  async getUsers(): Promise> {
    const response = await fetch(`${this.baseUrl}/users`);
    return response.json();
  }

  async getUserById(id: number): Promise> {
    const response = await fetch(`${this.baseUrl}/users/${id}`);
    return response.json();
  }

  async createUser(userData: CreateUserRequest): Promise> {
    const response = await fetch(`${this.baseUrl}/users`, {
      method: 'POST',
      headers: {
        'Content-Type': 'application/json',
      },
      body: JSON.stringify(userData),
    });
    return response.json();
  }

  async updateUser(id: number, userData: UpdateUserRequest): Promise> {
    const response = await fetch(`${this.baseUrl}/users/${id}`, {
      method: 'PUT',
      headers: {
        'Content-Type': 'application/json',
      },
      body: JSON.stringify(userData),
    });
    return response.json();
  }

  async deleteUser(id: number): Promise> {
    const response = await fetch(`${this.baseUrl}/users/${id}`, {
      method: 'DELETE',
    });
    return response.json();
  }
}

// Usage
const userService = new UserService();

// Type-safe API calls
const users = await userService.getUsers();
if (users.success) {
  console.log('Users:', users.data);
} else {
  console.error('Error:', users.errors);
}

Tips và Tricks

Type Assertions

// Khi bạn biết chắc type của một value
const userInput = document.getElementById('username') as HTMLInputElement;
const value = userInput.value; // TypeScript biết đây là string

Non-null Assertion

// Khi bạn chắc chắn value không null/undefined
const element = document.getElementById('myDiv')!;
element.classList.add('active');

Index Signatures

interface StringDictionary {
  [key: string]: string;
}

const config: StringDictionary = {
  apiUrl: 'https://api.example.com',
  timeout: '5000'
};
