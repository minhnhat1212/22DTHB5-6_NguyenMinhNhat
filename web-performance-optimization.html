Tối ưu hóa Performance cho Web Application

Kỹ thuật tối ưu tải trang và tương tác: lazy-loading, code-splitting, caching và quan trắc hiệu năng.

Tối ưu hóa performance là một trong những yếu tố quan trọng nhất quyết định trải nghiệm người dùng và thứ hạng SEO của website. Trong bài viết này, chúng ta sẽ khám phá các kỹ thuật và công cụ hiện đại để tối ưu hóa hiệu suất web application.

Core Web Vitals

Core Web Vitals là các chỉ số quan trọng nhất để đánh giá trải nghiệm người dùng trên web, được Google sử dụng để xếp hạng website.

LCP (Largest Contentful Paint)

Đo thời gian tải nội dung chính của trang. Mục tiêu: < 2.5s

Ví dụ: Thời gian hiển thị hero image hoặc tiêu đề chính

FID (First Input Delay)

Đo thời gian phản hồi khi người dùng tương tác lần đầu. Mục tiêu: < 100ms

Ví dụ: Thời gian từ khi click button đến khi có phản hồi

CLS (Cumulative Layout Shift)

Đo độ ổn định layout của trang. Mục tiêu: < 0.1

Ví dụ: Tránh layout shift khi load hình ảnh hoặc font

Đo Core Web Vitals với Performance Observer

// Đo LCP
new PerformanceObserver((list) => {
  for (const entry of list.getEntries()) {
    console.log('LCP:', entry.startTime);
    // Gửi dữ liệu đến analytics
    gtag('event', 'web_vitals', {
      name: 'LCP',
      value: Math.round(entry.startTime)
    });
  }
}).observe({ type: 'largest-contentful-paint', buffered: true });

// Đo CLS
let clsValue = 0;
new PerformanceObserver((list) => {
  for (const entry of list.getEntries()) {
    if (!entry.hadRecentInput) {
      clsValue += entry.value;
    }
  }
  console.log('CLS:', clsValue);
}).observe({ type: 'layout-shift', buffered: true });

Tối ưu hóa hình ảnh

Hình ảnh thường chiếm 60-80% dung lượng trang web. Tối ưu hóa hình ảnh là bước quan trọng để cải thiện performance.

1. Lazy Loading Images

Native Lazy Loading

<!-- HTML5 lazy loading -->
<img src="image.jpg" loading="lazy" alt="Description">

<!-- Intersection Observer cho browser cũ -->
<script>
const images = document.querySelectorAll('img[data-src]');
const imageObserver = new IntersectionObserver((entries, observer) => {
  entries.forEach(entry => {
    if (entry.isIntersecting) {
      const img = entry.target;
      img.src = img.dataset.src;
      img.classList.remove('lazy');
      imageObserver.unobserve(img);
    }
  });
});

images.forEach(img => imageObserver.observe(img));
</script>

3. Image Compression

Sharp.js cho server-side optimization

const sharp = require('sharp');

// Resize và compress
await sharp('input.jpg')
  .resize(800, 600)
  .jpeg({ quality: 80, progressive: true })
  .webp({ quality: 80 })
  .toFile('output.webp');

// Generate multiple sizes
const sizes = [400, 800, 1200];
for (const size of sizes) {
  await sharp('input.jpg')
    .resize(size)
    .jpeg({ quality: 80 })
    .toFile(`output-${size}w.jpg`);
}

Chiến lược Caching

Caching là kỹ thuật quan trọng để giảm thời gian tải và tiết kiệm băng thông.

1. HTTP Caching

Cache Headers

// Express.js cache headers
app.use(express.static('public', {
  maxAge: '1y', // Cache 1 năm cho static assets
  etag: true,
  lastModified: true
}));

// Cache API responses
app.get('/api/data', (req, res) => {
  res.set({
    'Cache-Control': 'public, max-age=3600', // 1 giờ
    'ETag': generateETag(data)
  });
  res.json(data);
});

2. Service Worker Caching

Advanced Service Worker

const CACHE_NAME = 'app-v1';
const STATIC_CACHE = 'static-v1';
const DYNAMIC_CACHE = 'dynamic-v1';

// Install event
self.addEventListener('install', event => {
  event.waitUntil(
    caches.open(STATIC_CACHE).then(cache => {
      return cache.addAll([
        '/',
        '/styles.css',
        '/script.js',
        '/manifest.json'
      ]);
    })
  );
});

// Fetch event với cache strategies
self.addEventListener('fetch', event => {
  const { request } = event;
  
  // Cache First cho static assets
  if (request.destination === 'image') {
    event.respondWith(cacheFirst(request, STATIC_CACHE));
  }
  // Network First cho API calls
  else if (request.url.includes('/api/')) {
    event.respondWith(networkFirst(request, DYNAMIC_CACHE));
  }
  // Stale While Revalidate cho HTML
  else {
    event.respondWith(staleWhileRevalidate(request, STATIC_CACHE));
  }
});

async function cacheFirst(request, cacheName) {
  const cached = await caches.match(request);
  return cached || fetch(request);
}

3. CDN và Edge Caching

Cloudflare Workers

addEventListener('fetch', event => {
  event.respondWith(handleRequest(event.request));
});

async function handleRequest(request) {
  const url = new URL(request.url);
  
  // Cache static assets at edge
  if (url.pathname.match(/\.(css|js|png|jpg|jpeg|gif|svg)$/)) {
    return caches.default.match(request).then(response => {
      if (response) {
        return response;
      }
      return fetch(request).then(fetchResponse => {
        return caches.default.put(request, fetchResponse.clone()).then(() => {
          return fetchResponse;
        });
      });
    });
  }
  
  return fetch(request);
}

Lazy Loading

Lazy loading giúp tải nội dung chỉ khi cần thiết, cải thiện đáng kể thời gian tải trang ban đầu.

1. Component Lazy Loading

React Lazy với Error Boundary

import React, { Suspense, lazy } from 'react';

const LazyComponent = lazy(() => import('./HeavyComponent'));

function App() {
  return (
    <ErrorBoundary fallback={<ErrorFallback />}>
      <Suspense fallback={<LoadingSpinner />}>
        <LazyComponent />
      </Suspense>
    </ErrorBoundary>
  );
}

// Error Boundary Component
class ErrorBoundary extends React.Component {
  constructor(props) {
    super(props);
    this.state = { hasError: false };
  }

  static getDerivedStateFromError(error) {
    return { hasError: true };
  }

  render() {
    if (this.state.hasError) {
      return this.props.fallback;
    }
    return this.props.children;
  }
}

Giám sát Performance

Giám sát liên tục performance giúp phát hiện và khắc phục các vấn đề kịp thời.

1. Real User Monitoring (RUM)

Custom Performance Monitoring

class PerformanceMonitor {
  constructor() {
    this.metrics = {};
    this.init();
  }

  init() {
    // Đo Core Web Vitals
    this.measureLCP();
    this.measureFID();
    this.measureCLS();
    
    // Đo custom metrics
    this.measurePageLoad();
    this.measureAPIResponse();
  }

  measureLCP() {
    new PerformanceObserver((list) => {
      const entries = list.getEntries();
      const lastEntry = entries[entries.length - 1];
      this.metrics.lcp = lastEntry.startTime;
      this.sendMetrics();
    }).observe({ type: 'largest-contentful-paint', buffered: true });
  }

  measureAPIResponse() {
    const originalFetch = window.fetch;
    window.fetch = async (...args) => {
      const start = performance.now();
      const response = await originalFetch(...args);
      const duration = performance.now() - start;
      
      this.metrics.apiResponse = duration;
      return response;
    };
  }

  sendMetrics() {
    // Gửi metrics đến analytics service
    fetch('/api/metrics', {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify(this.metrics)
    });
  }
}

new PerformanceMonitor();

Công cụ và Best Practices

1. Development Tools

Chrome DevTools

  • Performance tab để phân tích runtime
  • Network tab để kiểm tra tải tài nguyên
  • Lighthouse để audit toàn diện

WebPageTest

  • Kiểm tra performance từ nhiều vị trí
  • Video recording của quá trình tải
  • Waterfall chart chi tiết

GTmetrix

  • Phân tích PageSpeed và YSlow
  • Đề xuất tối ưu hóa cụ thể
  • Monitoring liên tục

2. Performance Best Practices

Critical Rendering Path

  • Tối ưu hóa CSS delivery (inline critical CSS)
  • Defer non-critical JavaScript
  • Sử dụng preload cho tài nguyên quan trọng

Resource Optimization

  • Minify CSS, JS, HTML
  • Enable Gzip/Brotli compression
  • Sử dụng HTTP/2 và HTTP/3

User Experience

  • Implement skeleton screens
  • Sử dụng progressive loading
  • Optimize for mobile-first

3. Performance Checklist
