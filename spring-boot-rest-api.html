Spring Boot: Xây dựng RESTful API từ cơ bản đến nâng cao

Lộ trình từng bước xây dựng RESTful API với Spring Boot: khởi tạo dự án, tạo Entity/Repository/Service/Controller, xử lý lỗi, validation và bảo mật cơ bản.

Khởi tạo và cấu trúc dự án

Bắt đầu với Spring Initializr và các dependency tối thiểu: Spring Web, Spring Data JPA, Validation. Tổ chức mã theo layered architecture để dễ mở rộng và test.

Các bước chính

  1. Tạo Entity và mapping với JPA
  2. Repository kế thừa JpaRepository
  3. Service chứa business logic
  4. Controller exposed endpoints (GET/POST/PUT/DELETE)

Validation và Error Handling

Sử dụng Bean Validation (@NotBlank, @Email, @Size) kết hợp @Valid trong Controller. Tạo @ControllerAdvice để chuẩn hóa lỗi (status code, message, timestamp).

Controller mẫu với Validation

@RestController
@RequestMapping("/api/v1/users")
public class UserController {
    private final UserService userService;

    public UserController(UserService userService) {
        this.userService = userService;
    }

    @PostMapping
    public ResponseEntity<UserResponse> create(@Valid @RequestBody CreateUserRequest req) {
        UserResponse created = userService.createUser(req);
        return ResponseEntity.status(HttpStatus.CREATED).body(created);
    }
}

record CreateUserRequest(@NotBlank String username, @Email String email) {}

Bảo mật cơ bản

Áp dụng Spring Security cho authentication/authorization. Với dự án demo, có thể dùng HTTP Basic hoặc JWT đơn giản.

Cấu hình Spring Security đơn giản

@Configuration
@EnableWebSecurity
public class SecurityConfig {
    @Bean
    SecurityFilterChain filterChain(HttpSecurity http) throws Exception {
        http.csrf().disable()
            .authorizeHttpRequests(auth -> auth
                .requestMatchers("/api/v1/public/**").permitAll()
                .anyRequest().authenticated())
            .httpBasic();
        return http.build();
    }
}

Tips triển khai

  • Đặt prefix cho API: /api/v1
  • Chuẩn hóa response (data, pagination, errors)
  • Viết Integration Tests cho endpoints quan trọng

Lỗi thường gặp

  • N+1 query: dùng fetch join hoặc DTO projection
  • Validate thiếu: quên @Valid ở method param
  • Mapping sai status code: 200 thay vì 201/204 cho create/delete

Paging và Sorting

Repository với Pageable

public interface UserRepository extends JpaRepository {
  Page findByActiveTrue(Pageable pageable);
}

@GetMapping
public Page list(@PageableDefault(size = 20, sort = "createdAt", direction = Sort.Direction.DESC) Pageable pageable) {
  return userService.listActive(pageable);
}
